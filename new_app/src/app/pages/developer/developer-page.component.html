Data Access & Developer Guide

The TopCastles dataset is openly available as static JSON, an OpenAPI-described REST API, and an MCP server for direct AI assistant integration. No authentication, no rate limits, no sign-up required.

Static JSON Endpoints

All endpoints are pre-generated static files served directly from the same origin. No server-side processing. Responses are cacheable and work from any language or tool.

GET /api/index.json API index and country list

Returns the API manifest: total castle count, all available endpoints, and the full list of country slugs with castle counts.

{{ snippets['curlIndex'] }}
GET /api/castles.json All 1,000 castles

Complete dataset sorted by score_total descending. Includes all enrichment fields.

{{ snippets['curlAll'] }}
GET /api/top100.json Top 100 castles by score

The 100 highest-scoring castles. Useful when you only need the most prominent entries.

{{ snippets['curlTop100'] }}
GET /api/by-country/{slug}.json Castles for one country

Returns all castles in a specific country, sorted by score. Slugs are lowercase hyphenated country names — full list in /api/index.json.

Data Schema

Every castle record is a flat JSON object. Fields from Wikipedia and Wikidata enrichment are nullable — not all castles have a Wikipedia article or Wikidata entry.

For AI Agents & LLMs

TopCastles follows the llms.txt convention so any LLM can self-discover the dataset without manual configuration.

Discovery file

A machine-readable discovery file is served at /llms.txt. It describes the dataset, all endpoints, the Castle schema, and example query patterns.

open_in_new View /llms.txt

Recommended access pattern

  1. Discover: Fetch /llms.txt or /api/index.json to learn available endpoints and schema.
  2. Fetch data: Use the most specific endpoint. For country queries use /api/by-country/{slug}.json rather than loading all 1,000 castles.
  3. Filter client-side: Era, condition, and type filtering is done after fetching — the static API has no query parameters.
  4. Or use MCP: The MCP server provides server-side filtering via search_castles — no client-side logic needed.
